### 🎬 使用流程演示

#### 1️⃣ 基础密码生成流程
```
用户操作步骤：
1. 打开网页 → 显示默认设置 (12字符，所有类型启用)
2. 查看预生成密码 → 显示强度等级
3. 点击"Generate New Password" → 立即生成新密码
4. 点击"Copy"按钮 → 显示"Copied!"提示，密码已复制
```

#### 2️⃣ 自定义密码配置流程
```
用户操作步骤：
1. 调整长度滑块 → 实时显示新长度数值
2. 关闭"大写字母"开关 → 密码重新生成，强度可能下降
3. 设置为仅数字模式 → 强度显示为"Weak"，显示安全建议
4. 调整到20字符长度 → 强度提升，显示"Very Strong"
```

#### 3️⃣ 主题切换演示
```
用户操作步骤：
1. 点击主题按钮 → 显示下拉菜单 (Light/Dark/System)
2. 选择"Dark" → 页面平滑切换到暗色主题
3. 选择"System" → 根据系统设置自动适应主题
4. 刷新页面 → 主题偏好已保存，维持选择
```

#### 4️⃣ 语言切换演示
```
用户操作步骤：
1. 点击语言按钮 → 显示16种语言列表
2. 选择"简体中文" → 所有界面文本切换为中文
3. 选择"العربية" → 界面切换为阿拉伯语，布局变为RTL
4. 关闭浏览器重新打开 → 语言偏好已保存
```

## 🏗️ 架构设计

### 组件分离策略
遵循Next.js 14最佳实践，严格分离服务器组件和客户端组件：

- **服务器组件** (`page.tsx`, `layout.tsx`): 
  - 处理SEO metadata
  - 静态内容渲染
  - 主题和i18n提供者包装
  - 不包含交互逻辑

- **客户端组件** (`password-generator-content.tsx`等):
  - 所有用户交互逻辑
  - 状态管理 (useState, useCallback)
  - 浏览器API调用 (clipboard API)
  - 主题切换和语言切换功能

### 状态管理
- **React内置hooks**: 密码选项、生成状态、复制状态
- **next-themes**: 主题状态管理（支持SSR无闪烁）
- **i18next**: 多语言状态管理和翻译
- **js-cookie**: 持久化语言偏好设置

### 国际化架构
- **配置分离**: 语言列表和翻译资源分别管理
- **按需加载**: 翻译资源统一加载（优化包大小）
- **类型安全**: TypeScript支持的语言代码类型检查
- **SSR友好**: 客户端初始化避免hydration mismatch

## 🔒 安全算法

### 密码生成算法
1. **字符集构建**: 根据用户选择构建可用字符集
2. **类型保证**: 确保每种选中的字符类型至少包含一个字符
3. **随机填充**: 使用`Math.random()`填充剩余长度
4. **字符洗牌**: Fisher-Yates算法打乱字符顺序

### 强度评估算法
基于以下因素进行评分：
- 密码长度 (8+, 12+, 16+ 字符)
- 字符类型多样性 (大小写、数字、符号)
- 综合评分映射到强度等级

## 🤝 贡献指南

### 代码规范
- **注释语言**: 代码注释必须使用中文
- **网页文案**: 用户界面文案必须使用英文
- **组件命名**: 使用PascalCase，文件名使用kebab-case
- **提交信息**: 使用中文描述具体改动

### 开发规则
- 遵循Next.js 14 App Router最佳实践
- 严格分离服务器组件和客户端组件
- 优先使用服务器组件，仅在必要时使用客户端组件
- 所有样式使用Tailwind CSS
